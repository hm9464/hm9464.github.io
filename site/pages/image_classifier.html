Image Classification of Cars with Convolutional Neural Networks

Jupter notebooks for all data loading, parsing, and analysis can be found here

Background

A convolutional neural networks is a powerful deep learning algorithm that is popularly used for image recognition and classification. In this project, I have built a convolutional neural network that is trained on determining what kind of make and model a given car image is closest to.

Convolutional neural networks are particularly useful for image recognition because they provide the ability to extract specific features out of images (feature detectors). We can then pool and flatten these feature sets to make these features machine-readable.

In this project, I will attempt to build 2 models: 1. Determine the make and model year of a given car 2. Determine the make only of a given car

I decided to build the second, more general model to see if it was possible to classify cars just on make, since there would be more instances for a given make. In addition, I was curious to see if it would work well, since car manufacturers tend to have their own 'style' when designing a car. For example, BMW has their signature double grill front that they have sported on almost all their cars in the last 20 years.

Data Source

Stanford AI Cars database: The data contains 196 classes of cars with their make and model (e.g. 2007 Toyota Corolla). The data is split into an almost even training and testing sample, with 8144 images for training and 8000 for testing.

2. Model Building and Evaluation

I started by running a simple model with the following parameters:

Parameter Value
img_pixels 256x256
n_filters 512
layer_nodes 400
batchsize 616
epochs 50
steps_per_epoch 100
validation_steps 50
pool_size (2,2)
kernel_size (3,3)
dropout 0.2

However, over the course of 30 epochs, the model accuracy didn't seem to budge from ~0.08 (8%). I interpreted this to mean that the model was not learning over time, regardless of the batch size and number of epochs that the training was iterating over.

Therefore, I decided to increase the number of image pixels to 200x200, since the model may not have been able to detect any features due to pixellation.

Below are plots detailing the accuracy of the model with 200x200 pixels.

While training accuracy seems to improve over time as the loss reduces, the validation accuracy stays relatively constant while loss initially decreases, before increasing after about the 20th epoch. This could be indicative of overfitting.
